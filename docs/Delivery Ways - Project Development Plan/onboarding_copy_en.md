**Purpose**: Localization content for onboarding and in-app guidance (Sale-Only compliant)

---

## Guidelines

1. **Sale-Only Compliance**: All text must be truthful about feature availability.
2. **Conditional Language**: Use phrases like "when available" for backend-dependent features.
3. **No False Promises**: Never promise features that are not implemented or enabled.
4. **Clear & Concise**: Keep text short and actionable.
